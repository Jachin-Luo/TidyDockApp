# TidyDock v0.2.0 Development Plan

## Goal

TidyDock v0.2.0 is a Windows-only WinForms rewrite focused on lower resident resource usage and a cleaner architecture. The WPF v0.1 implementation remains in `src/TidyDock` as a legacy reference.

The release should focus on three themes:

- Lower-resource Dock runtime.
- Cleaner service/UI separation.
- A self-contained v0.2 config and packaging flow.

## Scope

### WinForms Rewrite

- Add `src/TidyDock.WinForms` targeting `net8.0-windows`.
- Use WinForms plus Win32 interop for the Dock window, tray, hot zone, startup, and Alt+Tab hiding.
- Store v0.2 data under `%APPDATA%\TidyDock\winforms`.
- Do not migrate v0.1 WPF config.

### MVP Parity

- App, file, folder, URL, separator, and settings Dock items.
- Drag files/folders into the Dock.
- Drag Dock items to reorder.
- Right-click menu and tray menu.
- Settings window with explicit dark/light colors.
- Auto-hide hot zone and Show Desktop recovery.

### Performance Observability

- Record startup time in local logs.
- Keep memory measurement repeatable.
- Add a simple performance baseline update process for each release.
- Compare WinForms v0.2 measurements against WPF v0.1.1.

## Out of Scope

- Automatic desktop cleanup.
- Running app indicators.
- Window switching.
- Cloud sync.
- File indexing.
- Auto update.
- Signed installer.

## Acceptance Criteria

- v0.2 starts from a clean config path without touching v0.1 data.
- Build and portable verification pass.
- Release checklist is updated before tagging.
- Memory baseline is measured and compared with v0.1.1.

## Suggested Milestone

- Name: `v0.2.0`
- Target: next functional hardening release.
- Release type: minor.
- Branch strategy: small feature branches from `master`, merged after build verification.
