# TidyDock v0.2.0 开发计划

## 目标

TidyDock v0.2.0 是 Windows-only 的 WinForms 重写版本，重点是降低常驻资源占用并梳理更清晰的架构。WPF v0.1 实现保留在 `src/TidyDock`，作为 legacy 参考。

这个版本聚焦三件事：

- 更低资源占用的 Dock 运行时。
- 更清晰的 Service/UI 分层。
- 独立的 v0.2 配置和打包流程。

## 范围

### WinForms 重写

- 新增 `src/TidyDock.WinForms`，目标框架为 `net8.0-windows`。
- Dock 窗口、托盘、热区、开机启动和 Alt+Tab 隐藏使用 WinForms + Win32 interop。
- v0.2 数据保存到 `%APPDATA%\TidyDock\winforms`。
- 不迁移 v0.1 WPF 配置。

### MVP 对齐

- 支持应用、文件、文件夹、网址、分隔符和设置 Dock 项。
- 支持拖拽文件/文件夹到 Dock。
- 支持拖拽 Dock 项排序。
- 支持右键菜单和托盘菜单。
- 文件夹 Dock 项打开轻量文件夹面板，支持进入子文件夹、返回和在资源管理器中打开。
- 设置窗口显式处理深色/浅色配色。
- 支持自动隐藏热区和“显示桌面”后的恢复。

### 性能可观测性

- 在本地日志中记录启动耗时。
- 保持内存测量可重复。
- 为每个版本保留简单的性能基线更新流程。
- 将 WinForms v0.2 测量结果与 WPF v0.1.1 对比。

### UI 方向

- Dock 采用透明玻璃底座、细描边、柔和阴影和悬停放大。
- 设置页采用半透明背景，但文字和控件保持清晰可读。
- 文件夹面板采用轻量网格 Flyout，只读取直接子项，不递归扫描。
- 应用内可见文字默认使用中文；配置枚举值仍保持英文以保证兼容。
- 不引入重 UI 框架，继续使用轻量 WinForms/GDI+/Win32。

## 不做

- 自动桌面清理。
- 运行中应用指示器。
- 窗口切换。
- 云同步。
- 文件索引。
- 运行状态、小圆点、badge、窗口切换或进程监控。
- 自动更新。
- 签名安装包。

## 验收标准

- v0.2 从独立配置路径启动，不触碰 v0.1 数据。
- 构建和 portable 验证通过。
- 打 tag 前更新 release checklist。
- 测量内存基线，并与 v0.1.1 对比。

## 建议里程碑

- 名称：`v0.2.0`
- 目标：下一个功能硬化版本。
- 发布类型：minor。
- 分支策略：从 `master` 拉小功能分支，构建验证通过后合并。
