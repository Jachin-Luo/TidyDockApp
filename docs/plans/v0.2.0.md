# TidyDock v0.2.0 Development Plan

## Goal

TidyDock v0.2.0 should make the current MVP easier to maintain and safer to use without expanding the product into a taskbar replacement or desktop automation tool.

The release should focus on three themes:

- Better Dock item management.
- Better stability and recovery.
- Better performance observability.

## Scope

### Dock Item Management

- Add a right-click context menu for common Dock item actions.
- Add a broken-item state and relink flow.
- Improve shortcut import and migration diagnostics.
- Keep drag-to-reorder behavior stable.

### Stability and Recovery

- Add config validation before loading user data into the UI.
- Add config repair for missing or invalid required fields.
- Add clearer access to local logs.
- Add a backup and restore path for the user config.

### Performance Observability

- Record startup time in local logs.
- Keep memory measurement repeatable.
- Add a simple performance baseline update process for each release.
- Avoid broad framework migration until measurements justify it.

## Out of Scope

- Automatic desktop cleanup.
- Running app indicators.
- Window switching.
- Cloud sync.
- File indexing.
- Auto update.
- Signed installer.

## Acceptance Criteria

- Existing v0.1.1 user data loads without manual migration.
- Broken items are visible and recoverable instead of failing silently.
- Config corruption has a predictable fallback path.
- Build and portable verification pass.
- Release checklist is updated before tagging.
- Memory baseline is measured and compared with v0.1.1.

## Suggested Milestone

- Name: `v0.2.0`
- Target: next functional hardening release.
- Release type: minor.
- Branch strategy: small feature branches from `master`, merged after build verification.
