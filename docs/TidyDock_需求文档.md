# TidyDock 需求文档

版本：v1.1  
日期：2026-06-27  
平台：Windows 10 / Windows 11

## 1. 项目概述

TidyDock 是一款轻量 Windows 桌面 Dock 工具，用于美化桌面并收纳用户手动添加的常用入口。产品参考 macOS Dock 的视觉效果和文件夹 Stack 交互，但不替代 Windows 任务栏，不扫描桌面，不自动整理文件，不监控应用状态，不做后台索引。

第一阶段目标是：低内存、低 CPU、纯手动、本地保存、用户文件安全。

## 2. 产品边界

必须坚持：

- 不扫描桌面
- 不自动分类整理文件
- 不展示最近文件
- 不展示运行状态小圆点
- 不提供窗口列表
- 不做任务切换
- 不监控应用状态
- 不做后台索引
- 不联网
- 文件夹内容只在用户点击时按需读取

## 3. 核心功能

- 显示一个连续 Dock 栏，不默认拆分应用区、文件夹区或最近项目区
- 支持用户手动添加应用、快捷方式、文件夹、文件、URL 和分隔符
- 支持编辑模式，只有编辑模式下允许添加、删除、排序、重命名、修改目标和更换图标
- 支持拖入 `.exe`、`.lnk`、文件、文件夹到 Dock
- 支持拖拽调整 Dock 项目顺序，顺序不因最近任务变化而改变
- 支持点击应用、文件、URL 后使用系统默认方式打开
- 支持点击文件夹后弹出可滚动内容面板
- 支持文件夹面板进入子文件夹、返回上一级、在资源管理器中打开
- 支持文件夹面板展示文件/文件夹图标
- 支持 Dock 图标 hover 放大
- 支持完全隐藏 Dock 背景
- 支持图标名称显示开关
- 支持自动隐藏、始终置顶、启动显示、托盘入口、开机启动
- 支持配置目录、日志目录、图标缓存清理、重置配置等维护入口

## 4. 文件夹面板要求

文件夹面板只作为快速访问入口，不是文件管理器。

读取规则：

- 只读取当前文件夹的直接子项
- 不递归读取子目录
- 不读取文件内容
- 不生成图片或视频缩略图
- 默认不显示隐藏文件
- 默认最多展示 300 个项目
- 超出数量时展示提示
- 面板关闭后释放临时列表数据
- 快速切换目录时，只允许最后一次读取结果更新 UI

交互规则：

- 文件夹在前，文件在后
- 同类项目按名称排序
- 点击文件使用系统默认程序打开
- 点击文件夹进入该子目录
- 支持返回上一级
- 支持 Esc 关闭
- 鼠标离开后快速自动隐藏

## 5. 设置要求

外观设置：

- Dock 位置：底部、顶部、左侧、右侧
- 显示器选择
- 图标大小
- 图标间距
- hover 放大倍率
- 背景透明度，允许设置为 0
- 圆角
- 主题：跟随系统、浅色、深色
- 图标名称显示开关

行为设置：

- 编辑模式
- 自动隐藏
- 始终置顶
- 启动时显示 Dock
- 显示托盘图标
- 开机启动
- 语言：简体中文、英文
- 文件夹面板最大展示数量和最大高度
- 是否显示隐藏文件

维护设置：

- 打开配置目录
- 打开日志目录
- 清理图标缓存
- 重置配置
- 关于
- 退出程序

## 6. 数据存储

所有数据保存在本地：

```text
%APPDATA%\TidyDock\
  config\settings.json
  config\settings.json.bak
  cache\icons\
  logs\error.log
```

配置保存要求：

- 使用临时文件写入
- 写入完成后替换正式配置
- 替换时保留 `.bak` 备份
- 设置滑杆连续变化时应防抖保存，避免频繁磁盘写入
- 配置损坏时备份损坏文件并恢复默认配置

## 7. 稳定性要求

- 使用稳定产品级 Mutex 名称，防止重复运行实例
- 不使用带开发机编号或用户编号的 Mutex 名称
- 输入对话框按钮文案必须走本地化
- 未处理异常写入本地日志
- 移除 Dock 项目不删除原始文件
- 清理图标缓存不影响用户原始文件
- 退出程序前应刷新待保存配置

## 8. 性能要求

- 空闲 CPU 接近 0%
- 空闲状态不扫描文件系统
- 空闲状态不监控进程
- 不使用高频后台定时器驱动 UI
- 大文件夹打开时 UI 不明显卡顿
- 图标优先使用 Windows 高分辨率系统图标，避免放大后模糊
- 设置变更时减少重复写盘

## 9. 验收标准

- 可以添加并打开 `.exe`、`.lnk`、文件、文件夹和 URL
- 关闭编辑模式后不能添加、删除、排序、重命名或更换图标
- Dock 顺序始终保持用户手动顺序
- 文件夹弹窗展示文件/文件夹图标
- 快速进入多个文件夹时不会被旧读取结果覆盖
- 输入框按钮在中文/英文模式下显示正确
- 重复启动应用时只保留一个进程
- 配置保存后生成或更新 `.bak`
- 拖动设置滑杆时不会频繁写入配置文件
- 构建脚本和便携包验证脚本通过
