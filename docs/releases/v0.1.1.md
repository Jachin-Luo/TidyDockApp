# TidyDock v0.1.1 Release Notes

TidyDock v0.1.1 is the first stable MVP baseline after the project structure cleanup, documentation pass, and post-MVP behavior fixes. It keeps the product scope intentionally small: a lightweight Windows desktop Dock for manually organizing frequently used apps, files, folders, URLs, and separators without desktop scanning or background indexing.

## Highlights

- Manual Dock item management for apps, shortcuts, folders, files, URLs, and separators.
- Edit mode for adding, editing, deleting, and reordering Dock items.
- Folder panel for quick folder access.
- Settings window for theme, display, startup, language, labels, opacity, tray, and maintenance actions.
- Local JSON configuration, icon cache, error log, and backup-assisted config saving.
- Portable package and current-user installer.
- Dock and hot zone are hidden from Alt+Tab.
- Dock restores itself after Windows Show Desktop minimizes it.
- `.lnk` shortcuts are imported into the local TidyDock data directory so Dock items do not depend on desktop shortcut files.
- Icon loading now decodes custom bitmap icons near display size, reducing idle private memory from about 101.69 MB to 72.29 MB in the measured scenario.

```powershell
powershell -ExecutionPolicy Bypass -File '.\src\TidyDock\scripts\Build-Installer.ps1'
powershell -ExecutionPolicy Bypass -File '.\src\TidyDock\scripts\Test-Portable.ps1' -SkipLaunch
```

## Explicit Non-Goals

- Desktop scanning.
- Automatic file organization.
- Recent files.
- App running-state indicators.
- Window list or task switching.
- Process monitoring.
- Background indexing.
- Network behavior.

## Known Follow-Up Areas

- Better broken item recovery and relinking.
- More visible error/log entry points.
- More repeatable performance and startup measurements.
- More automation around GitHub Releases.
