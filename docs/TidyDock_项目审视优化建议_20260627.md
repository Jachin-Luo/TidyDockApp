## 高优先级建议

1. 修复输入对话框按钮文案

   文件：`TidyDockApp/InputDialog.cs`

   按钮文案应接入 `LocalizationService`，支持 `zh-CN` 和 `en-US`。

2. 修正单实例 Mutex 名称

   文件：`TidyDockApp/App.xaml.cs`

   Mutex 名称应使用稳定产品级名称，避免残留开发机或用户编号。

3. 强化配置文件保存可靠性

   文件：`TidyDockApp/SettingsService.cs`

   保存流程应使用临时文件和 `.bak` 备份，优先使用 `File.Replace`，降低配置中途损坏风险。

4. 避免文件夹面板异步读取竞争

   文件：`TidyDockApp/FolderPanel.cs`

   快速进入多个目录时，旧读取结果不应覆盖新目录 UI。可使用递增版本号或取消令牌。

5. 减少设置变更时的频繁写盘

   文件：`TidyDockApp/MainWindow.cs`、`TidyDockApp/SettingsWindow.cs`

   设置窗口滑杆拖动时应立即应用视觉变化，但配置保存应防抖。

## 中优先级建议

1. 设置窗口主题统一

   当前设置窗口按产品要求保持白底黑字，后续如需要可再统一到 `ThemeService`。

2. 重新评估启动时内存裁剪

   当前保留启动空闲期 working-set trim。后续应基于实测决定是否继续保留。

3. 评估升级目标框架

   当前仍为 `.NET Framework 4.6.1`。后续可评估升级到 `.NET Framework 4.8` 或现代 .NET Windows Desktop。

4. 增强异常记录

   用户无感失败可以继续保持静默体验，但关键失败应写入本地日志，便于排查。

## 结构优化建议

后续可以逐步拆分：

- `DockItemService`：新增、删除、重命名、移动、目标编辑
- `DockMenuBuilder`：右键菜单与 Dock 菜单构建
- `DockLayoutService`：窗口定位、方向、热区计算
- `DockRenderer`：Dock item UI 构建

- 输入对话框 OK / Cancel 按钮接入本地化
