# TidyDock 设计文档

版本：v1.1  
日期：2026-06-27  
平台：Windows 10 / Windows 11

## 1. 设计目标

TidyDock 是一个轻量桌面 Dock，而不是任务栏替代品、文件管理器或自动整理工具。第一阶段的设计目标是少做功能，把 Dock 本身做轻、做稳、做顺。

核心原则：

- 用户完全手动管理 Dock 项目
- 默认不扫描、不索引、不监控、不联网
- 常驻资源占用尽量低
- 所有配置和缓存都在本地
- 用户文件绝对安全

## 2. 整体结构

主要模块：

- `App.xaml.cs`：应用启动、单实例控制、全局异常处理、退出清理
- `MainWindow.cs`：Dock 主窗口、项目渲染、拖拽、菜单、定位和设置应用
- `FolderPanel.cs`：文件夹弹窗、异步目录读取、面板导航
- `SettingsWindow.cs`：设置窗口和 Dock 项目管理
- `SettingsService.cs`：配置读写、目录初始化、图标缓存清理
- `IconCacheService.cs`：系统图标提取、图标缓存、自定义图标加载
- `LocalizationService.cs`：中英文界面文案
- `ThemeService.cs`：主题颜色
- `TrayService.cs`：托盘图标和托盘菜单
- `StartupService.cs`：开机启动注册表入口
- `ErrorLogService.cs`：本地异常日志

后续如果继续扩展，可逐步拆分：

- `DockItemService`：新增、删除、重命名、移动、目标编辑
- `DockMenuBuilder`：右键菜单和 Dock 菜单构建
- `DockLayoutService`：窗口定位、方向、热区计算
- `DockRenderer`：Dock 项 UI 构建

## 3. Dock 主窗口设计

Dock 是一个无边框透明 WPF 窗口，默认显示在屏幕底部居中。用户可以选择底部、顶部、左侧、右侧。

设计规则：

- Dock 是连续容器，不默认分区
- 分隔符只是用户手动添加的视觉分组
- Dock 背景透明度可以设置为 0
- 始终置顶默认关闭，避免遮挡其他应用
- Hover 放大只作用于当前图标，不使用持续动画循环
- 图标名称由用户开关控制
- 编辑模式关闭时，拖入、排序、删除、重命名、更换图标全部禁用

## 4. 文件夹面板设计

文件夹面板模仿 macOS Dock Stack 的快速访问体验，但保持轻量。

面板结构：

- 顶部工具栏
- 返回按钮
- 当前目录标题
- 在资源管理器中打开按钮
- 关闭按钮
- 可滚动内容区

读取策略：

- 用户点击文件夹时才读取
- 使用异步任务读取目录，避免阻塞 UI
- 只读取直接子项，不递归
- 读取数量受 `MaxItems` 限制
- 每次读取带版本号，只有最后一次读取可以更新 UI
- 面板关闭时递增版本号并清空列表，防止旧异步结果回写

展示策略：

- 每行左侧展示 18px 文件/文件夹系统图标
- 文件夹名称加粗
- 文件夹在前，文件在后，同类按名称排序
- 不生成缩略图
- 不缓存目录列表

## 5. 图标设计

图标质量直接影响 Dock 观感。

策略：

- Dock 主图标优先从 Windows 系统图像列表读取高分辨率图标
- 64px 及以上优先使用 `SHIL_JUMBO`
- 较小图标使用 `SHIL_EXTRALARGE`
- 提取失败时回退到 `SHGetFileInfo`
- 缓存键包含版本和目标尺寸，避免旧低清缓存复用
- 文件夹面板中的文件类型图标按需获取并缓存
- 用户可在编辑模式下自定义图标

## 6. 配置保存设计

配置文件保存在：

```text
%APPDATA%\TidyDock\config\settings.json
```

保存流程：

1. 写入 `settings.json.tmp`
2. 刷新文件内容到磁盘
3. 如果正式配置已存在，使用 `File.Replace` 替换
4. 替换时保留 `settings.json.bak`
5. 如果 `File.Replace` 不可用，先复制旧配置为 `.bak`，再移动新配置

设置窗口中的滑杆和选项会立即应用视觉变化，但保存使用 450ms 防抖，减少连续拖动时的重复磁盘写入。退出应用时会刷新待保存配置。

## 7. 单实例设计

应用启动时创建稳定产品级 Mutex：

```text
Local\TidyDock.{83B8BC34-3B76-4E99-99D6-6D4F8D414B52}
```

如果 Mutex 已存在，新的进程立即退出，避免多个 Dock 同时运行。

## 8. 本地化设计

`LocalizationService` 提供核心 UI 文案。当前支持：

- `zh-CN`
- `en-US`

要求：

- 主菜单、右键菜单、设置项、文件夹面板提示均走本地化
- 输入对话框的 OK / Cancel 按钮也走本地化
- 新增 UI 文案时必须同步补齐中英文

## 9. 设置窗口设计

设置窗口保持白底黑字，优先简单清晰。它包含：

- 外观设置
- 行为设置
- Dock 项目管理
- 维护操作

设置窗口中的项目管理同样受编辑模式约束。关闭编辑模式后不能新增、移动、重命名、修改目标、更换图标或删除 Dock 项目。

## 10. 性能设计

性能边界：

- 空闲时不扫描桌面
- 空闲时不读取文件夹
- 空闲时不监控进程
- 不维护最近文件
- 不做窗口枚举
- 不做后台索引
- 不使用高频动画定时器
- 设置变更写盘防抖
- 文件夹读取异步且限制数量

## 11. 发布验证

发布前必须执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\TidyDockApp\scripts\Build-Installer.ps1
powershell -ExecutionPolicy Bypass -File .\TidyDockApp\scripts\Test-Portable.ps1 -SkipLaunch
```

人工烟测：

- 安装和卸载
- 单实例行为
- 配置保存和 `.bak` 生成
- 文件夹弹窗进入/返回/自动隐藏
- 多显示器定位
- 中文和英文输入框按钮显示
- README、需求文档、设计文档无乱码

## 12. 不做项

第一阶段明确不做：

- 桌面自动扫描
- 自动分类整理文件
- 最近文件
- 运行状态小圆点
- 窗口列表
- 任务切换
- 应用状态监控
- 后台索引
- 复杂规则引擎
- 文件内容预览
- 图片/视频缩略图生成
- 云同步
