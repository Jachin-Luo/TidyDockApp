# Security Policy

## Supported Versions

TidyDock is in early development. Security fixes are handled on the current development line unless a stable release branch is created later.

## Reporting A Vulnerability

Please do not publish sensitive security details in a public issue.

If this repository is hosted on GitHub, use GitHub private vulnerability reporting when available. Otherwise, open a public issue with a minimal description and ask for a private contact path.

## Privacy Boundary

TidyDock is intended to be local-first:

- No analytics.
- No account system.
- No cloud sync.
- No desktop scan.
- No process monitoring.
- No background indexing.
- No network behavior in the MVP.

See [docs/11-security-privacy.md](docs/11-security-privacy.md) for the detailed privacy model.
