# TidyDock 需求文档

版本：v1.0  
日期：2026-06-27  
平台：Windows 10 / Windows 11

## 1. 项目概述

TidyDock 是一款面向 Windows 的轻量桌面 Dock 工具，用于美化桌面，并收纳用户手动添加的常用应用、快捷方式、文件夹、文件和网页入口。

产品参考 macOS Dock 的视觉效果和文件夹 Stack 交互，但不替代 Windows 任务栏，不扫描桌面文件，不自动整理文件，不监控应用状态，不做后台索引。

第一版必须坚持低内存、低 CPU、纯手动、本地保存、用户文件安全。

## 2. 产品定位

一句话定位：

> 一个低占用、纯手动配置的 Windows 桌面 Dock，用来美化桌面和收纳快捷入口。

核心原则：

- 不替代系统任务栏。
- 不扫描桌面。
- 不自动整理文件。
- 不监控进程。
- 不后台索引。
- 不联网。
- 文件夹内容只在用户点击时按需读取。
- 所有配置保存在本地。

## 3. 目标用户

主要用户：

- 希望桌面更美观、更整洁的 Windows 用户。
- 常用软件较多，希望通过 Dock 快速启动的用户。
- 希望把常用文件夹集中放在桌面边缘的用户。
- 喜欢 macOS Dock 交互但仍使用 Windows 的用户。
- 对常驻软件性能占用敏感的用户。

非目标用户：

- 希望完全替代 Windows 任务栏的用户。
- 需要文件自动分类、自动整理、自动归档的用户。
- 需要复杂窗口管理或任务切换的用户。
- 需要最近文件、应用运行状态、窗口列表的用户。

## 5. Dock 主栏需求

Dock 主栏是软件核心界面，常驻在屏幕边缘，用于展示用户手动添加的快捷入口。

功能需求：

- 默认位于屏幕底部居中。
- 支持底部、顶部、左侧、右侧四个位置。
- Dock 是连续容器，不默认分区。
- 支持设置是否始终置顶。
- 支持设置是否自动隐藏。
- 编辑模式开启时，支持拖拽移动 Dock 项目顺序。
- 编辑模式开启时，支持拖入文件、文件夹、`.exe`、`.lnk` 添加项目。
- 支持空 Dock 状态下的添加入口。
- 支持托盘菜单唤出设置。

视觉需求：

- 半透明背景，透明度可降为 0 以完全隐藏 Dock 背景。
- 圆角可配置。
- 图标排列整齐，间距稳定。
- 鼠标悬停时图标放大，附近图标可轻微联动。
- 动画平滑但克制，不能造成明显 CPU 占用。

## 6. Dock 项目类型

### 6.1 应用

- 支持 `.exe` 和 `.lnk`。
- 点击后启动应用。
- `.lnk` 按 Windows 快捷方式原有逻辑打开。
- 显示应用图标。
- 支持自定义显示名称和图标。

### 6.2 文件夹

- 点击后弹出文件夹内容面板。
- 支持自定义显示名称和图标。
- 右键菜单可选择在资源管理器中打开。
- 文件夹只是用户手动添加的普通 Dock 项目，不是独立区域，也不是自动生成入口。

### 6.3 文件

- 点击后使用 Windows 默认程序打开。
- 显示对应文件类型图标。
- 支持自定义显示名称和图标。

### 6.4 URL

- 点击后使用默认浏览器打开。
- 支持 `http://` 和 `https://`。
- 支持自定义名称和图标。

### 6.5 分隔符

- 仅用于视觉分组。
- 默认不显示，只有用户手动添加后才出现。
- 不响应打开行为。
- 可拖拽排序。

## 7. 添加和编辑需求

添加方式：

- 开启编辑模式后，拖拽 `.exe` 到 Dock。
- 开启编辑模式后，拖拽 `.lnk` 到 Dock。
- 开启编辑模式后，拖拽文件夹到 Dock。
- 开启编辑模式后，拖拽普通文件到 Dock。
- 开启编辑模式后，通过菜单手动添加 URL。
- 开启编辑模式后，通过菜单手动添加分隔符。
- 开启编辑模式后，通过设置面板添加应用、文件、文件夹、URL、分隔符。

添加后处理：

- 自动判断项目类型。
- 提取或生成默认图标。
- 写入本地配置。
- 将图标缓存到本地缓存目录。
- 添加完成后立即出现在 Dock 中。

右键能力：

- 打开。
- 在资源管理器中显示或打开。

编辑模式开启后额外支持：

- 编辑名称。
- 编辑目标路径或 URL。
- 更换图标。
- 从 Dock 移除。

移除规则：

- 只删除 Dock 配置项。
- 不删除原始文件。
- 不移动用户文件。

## 8. 文件夹内容面板需求

点击 Dock 中的文件夹项目后，弹出类似 macOS Dock Stack 的内容面板，用于快速浏览并打开文件夹中的内容。

第一版采用：

> 弹出式滚动网格。

展开方向：

- Dock 在底部时，面板向上弹出。
- Dock 在顶部时，面板向下弹出。
- Dock 在左侧时，面板向右弹出。
- Dock 在右侧时，面板向左弹出。

面板内容：

- 显示当前文件夹中的直接子项目。
- 显示文件夹和文件。
- 默认不显示隐藏文件。
- 不递归读取子目录。
- 不读取文件内容。
- 不生成图片或视频缩略图。

交互：

- 鼠标滚轮上下滚动。
- 点击文件：使用默认程序打开。
- 点击文件夹：进入该子文件夹。
- 顶部提供返回按钮。
- 顶部显示当前文件夹名称。
- 提供“在资源管理器中打开”按钮。
- 点击面板外部关闭。
- 按 `Esc` 关闭。

性能限制：

- 默认最多显示 300 个项目。
- 超出数量时不继续渲染全部内容。
- 显示项目过多提示。
- 文件夹图标和文件类型图标按需加载。
- 面板关闭后释放临时列表数据。
- 不缓存文件夹列表。

排序规则：

1. 文件夹在前。
2. 文件在后。
3. 同类型按名称排序。

## 9. 设置需求

常规设置：

- 编辑模式。
- 开机启动。
- 显示托盘图标。
- 启动时显示 Dock。
- 语言：简体中文、英文。
- 打开配置目录。
- 打开日志目录。
- 清理图标缓存。
- 重置配置。
- 退出程序。
- 关于。

Dock 外观：

- Dock 位置。
- 显示器选择。
- 图标基础大小。
- 图标放大倍率。
- 图标间距。
- 背景透明度。
- 背景透明度可设置为 0。
- 圆角大小。
- 主题：跟随系统、浅色、深色。

行为设置：

- 自动隐藏。
- 文件夹面板最大高度。
- 文件夹面板最大展示数量。
- 是否显示隐藏文件。

Dock 项目管理：

- 查看所有 Dock 项目。
- 调整顺序。
- 编辑名称。
- 编辑目标路径。
- 更换图标。
- 移除项目。

## 10. 托盘需求

托盘菜单包含：

- 显示 / 隐藏 Dock。
- 设置。
- 关于。
- 退出。

行为：

- 关闭设置窗口不退出程序。
- 隐藏 Dock 不退出程序。
- 退出程序需要通过托盘菜单或设置中的退出按钮触发。
- 如果启动时隐藏 Dock，应保留托盘图标作为入口。

## 11. 数据存储需求

配置保存为本地 JSON 文件。

数据目录：

```text
%APPDATA%\TidyDock\
  config\settings.json
  cache\icons\
  logs\error.log
```

存储原则：

- 配置保存在本地。
- Dock 项图标在添加时提取并缓存。
- 文件夹面板中的文件类型图标可按需缓存。
- 不缓存文件夹列表。
- 不缓存图片和视频缩略图。
- 错误日志只记录必要异常，不记录用户文件列表。

## 12. 非功能需求

### 12.1 性能

- 空闲状态不执行定时扫描。
- 空闲状态不主动读取文件系统。
- 空闲状态不监控进程。
- 空闲 CPU 接近 0%。
- 文件夹内容读取必须异步执行，避免 UI 卡顿。
- 大目录必须有数量限制。
- 首次默认配置不添加样例 Dock 项。

### 12.2 稳定性

- Dock 崩溃不应影响用户文件。
- 移除 Dock 项不删除原文件。
- 清理缓存不影响用户原始文件。
- 配置写入应尽量避免中途损坏，可使用临时文件替换策略。
- 未处理异常写入本地日志。

### 12.3 隐私

- 不联网。
- 不上传任何文件信息。
- 不读取用户未主动添加的目录。
- 不扫描桌面。
- 不统计用户应用使用情况。

### 12.4 兼容性

目标系统：

- Windows 10
- Windows 11

屏幕支持：

- 1080p
- 2K
- 4K
- 高 DPI 缩放
- 多显示器轻量支持

## 13. 技术建议

推荐技术栈：

- C#
- WPF
- 少量 Win32 API

原因：

- 常驻内存低于 Electron。
- 适合 Windows 桌面软件。
- 系统图标、快捷方式、托盘、开机启动支持成熟。
- 透明窗口和无边框窗口实现方便。

不推荐第一版使用 Electron：

- 常驻内存较高。
- 打包体积较大。
- 对轻量桌面工具不够友好。

## 14. 验收标准

### 14.1 功能验收

- 用户可以将 `.exe` 拖入 Dock 并点击启动。
- 用户可以将 `.lnk` 拖入 Dock 并点击打开。
- 用户可以将文件夹拖入 Dock。
- 用户可以将普通文件拖入 Dock。
- 用户可以手动添加 URL。
- 用户可以手动添加分隔符。
- 用户关闭编辑模式后，不能拖入添加、拖拽排序、改名、更换图标或移除 Dock 项。
- 用户开启编辑模式后，可以进行 Dock 项编辑操作。
- 点击文件夹后可以弹出滚动内容面板。
- 文件夹面板可以打开文件。
- 文件夹面板可以进入子文件夹并返回。
- 用户可以拖拽调整 Dock 项目顺序。
- 用户可以右键移除 Dock 项目。
- 用户可以更换 Dock 项目图标。
- 用户可以调整 Dock 位置、图标大小、透明度和圆角。
- 用户可以通过托盘或设置退出程序。

### 14.2 性能验收

- 软件空闲时 CPU 接近 0%。
- 软件启动时间目标小于 1 秒。
- 软件常驻内存控制在轻量桌面工具合理范围内。
- 点击大文件夹时 UI 不明显卡死。
- 文件夹面板关闭后释放临时列表数据。
- 软件不进行桌面扫描。
- 软件不监控正在运行的应用。
- 软件不产生网络请求。

### 14.3 安全与隐私验收

- 移除 Dock 项不会删除原文件。
- 软件不会自动移动用户文件。
- 软件不会读取未添加的文件夹内容。
- 文件夹内容只在点击时读取。
- 配置保存在本地。
- 无后台上传行为。

## 16. 产品边界总结

TidyDock 第一版要做得少，但做得轻、稳、好看。

它不是任务栏替代品，不是文件整理器，不是窗口管理器，也不是自动化工具。它的核心价值是：

- 美化桌面。
- 收纳快捷入口。
- 快速打开常用位置。
- 用低资源占用提供接近 macOS Dock 的体验。

第一版成功的关键不是功能数量，而是：

- 空闲时安静。
- 打开时迅速。
- 动画时顺滑。
- 用户文件绝对安全。
- 产品边界清晰。
