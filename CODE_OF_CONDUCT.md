# Code Of Conduct

## Our Standard

TidyDock should be a practical, respectful project space. Be direct, kind, and focused on improving the software.

Examples of welcome behavior:

- Clear bug reports and reproducible steps.
- Respectful disagreement.
- Small, reviewable pull requests.
- Patience with new contributors.

Examples of unacceptable behavior:

- Harassment or personal attacks.
- Insults, threats, or discriminatory language.
- Publishing private information.
- Repeatedly pushing off-scope changes after maintainers have declined them.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or contributions that violate this code of conduct.
